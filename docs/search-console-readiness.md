# Search Console Readiness Checklist

## Already implemented in the site
- Brand-first page titles on key pages
- Stronger meta descriptions on homepage, services, industries, and contact
- Canonical tags on generated routes
- FAQ schema on homepage, services, industries, and contact
- Organization, ProfessionalService, and page-level structured data
- Sitemap at `https://www.vextor.co/sitemap.xml`
- `robots.txt` pointing to the sitemap
- `llms.txt` and `llms-full.txt` for AI and answer-engine discoverability
- Contact/address/social consistency in structured data
- Static route generation for direct page reloads

## Search Console actions to complete manually
1. Verify the domain property for `vextor.co` and `www.vextor.co`.
2. Submit `https://www.vextor.co/sitemap.xml`.
3. Use URL Inspection on `/`, `/services`, `/industries`, `/about`, `/contact`, and `/work`.
4. Request indexing after major content updates.
5. Check Coverage for excluded or duplicate URLs.
6. Check Enhancements for structured data visibility.
7. Review Performance after 2-4 weeks for branded queries like `vextor`, `vextor salesforce`, and `vextor buildertek`.

## Brand SEO work still needed outside the repo
1. Create or optimize Google Business Profile with the Ahmedabad address.
2. Use the same company name and URL across LinkedIn, Instagram, Facebook, X, and business directories.
3. Publish company profiles and citations on relevant directories.
4. Earn backlinks from partners, client references, industry communities, and Salesforce-related ecosystems.

## LLM and answer-engine discoverability
1. Keep `llms.txt` accurate whenever services, contact details, or positioning change.
2. Maintain clear factual pages for About, Services, Industries, and Contact.
3. Keep structured data consistent with visible page copy.
4. Build external mentions and citations so AI systems see Vextor across the web, not only on its own domain.
5. Publish more evidence-rich content such as case studies, delivery notes, FAQs, and implementation guidance.
